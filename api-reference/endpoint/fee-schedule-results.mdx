## Overview

Returns a paginated list of fee schedule results with comprehensive filtering. Each result represents a single fee or rebate line item extracted from an exchange's fee schedule document.

Results are automatically scoped to exchanges associated with your API key.

<Note>
  **Authorization Required** - Include your API key in the
  `x-rulebook-api-access-key` header. Contact
  [aaron@rulebookcompany.com](mailto:aaron@rulebookcompany.com) to obtain
  access.
</Note>

<Tip>
  Use [Get Available Filters](/api-reference/endpoint/fee-schedule-filters) or
  [Get Exchange Details](/api-reference/endpoint/exchange-detail) to discover
  valid filter values before querying.
</Tip>

## Query Parameters

### Filters

All filter parameters accept multiple values using repeated query parameters (e.g., `?exchange_name=Nasdaq&exchange_name=fee_cboe_us_options`). Multiple values are combined with OR logic.

<ParamField query="exchange_name" type="string[]">
  Filter by exchange identifier(s) — use values from the `exchange_names` field returned by [Get Available Filters](/api-reference/endpoint/fee-schedule-filters)

**Example:** `?exchange_name=fee_cboe_us_options&exchange_name=Nasdaq`

</ParamField>

<ParamField query="fee_type" type="string[]">
  Filter by fee type(s): `option`, `equity`
</ParamField>

<ParamField query="fee_category" type="string[]">
  Filter by fee category(s): `fees_and_rebates`, `general_provisions`, `legal_regulatory_fee`,
  `market_data_fees`, `participant_fee`, `port_fees_and_other_services`
</ParamField>

<ParamField query="fee_action" type="string[]">
  Filter by trading action(s): `make`, `take`, `open`, `routed`, `other`
</ParamField>

<ParamField query="fee_participant" type="string[]">
  Filter by participant type(s): `customer`, `firm`, `broker_dealer`, `market_maker`,
  `away_market_maker`, `professional`, `other`, `all`
</ParamField>

<ParamField query="fee_symbol_classification" type="string[]">
  Filter by symbol classification(s): `etf`, `etn`, `equity_class`, `index`, `not_applicable`, `all`
</ParamField>

<ParamField query="fee_symbol_type" type="string[]">
  Filter by symbol type(s): `penny`, `non_penny`, `not_applicable`, `both`
</ParamField>

<ParamField query="fee_trade_type" type="string[]">
  Filter by trade type(s): `index_options`, `multiply_traded_options`, `simple_order`, `complex_order`
</ParamField>

### Date Range

<ParamField query="start_date" type="string">
  Filter results scraped on or after this date (format: `YYYY-MM-DD`)
</ParamField>

<ParamField query="end_date" type="string">
  Filter results scraped on or before this date (format: `YYYY-MM-DD`)
</ParamField>

### Versioning

<ParamField query="latest_only" type="boolean" default="false">
  When `true`, returns only the most recent version per unique (supplier,
  fee_type) combination. Use this to get only the current fee data without
  historical versions.
</ParamField>

### Sorting

<ParamField query="order_by" type="string" default="created_on">
  Field to sort by: `created_on`, `scraped_time`, `fee_type`, `fee_category`
</ParamField>

<ParamField query="order_dir" type="string" default="desc">
  Sort direction: `asc` or `desc`
</ParamField>

### Pagination

<ParamField query="page_size" type="integer" default="20">
  Records per page (1–100)
</ParamField>

<ParamField query="page_number" type="integer" default="0">
  Page number (0-indexed)
</ParamField>

### Relationships

<ParamField query="include_footnotes" type="boolean" default="false">
  When `true`, includes resolved footnotes from the source document in the `relationships` field of each result.
</ParamField>

<ParamField query="include_definitions" type="boolean" default="false">
  When `true`, includes resolved definitions from the source document in the `relationships` field of each result.
</ParamField>

### Source Document

<ParamField query="include_source" type="boolean" default="false">
  When `true`, includes `source_document`, `source_link`, and `page_number` fields in each result.
</ParamField>

## Request Examples

**Basic request — latest CBOE BZX options fees:**

```bash
curl -X GET "https://api.rulebookcompany.com/api/v1/fee-schedule-results?exchange_name=fee_cboe_us_bzx_options&fee_type=option&latest_only=true" \
  -H "x-rulebook-api-access-key: YOUR_API_KEY"
```

**Multi-exchange Market Maker Make fees:**

```bash
curl -X GET "https://api.rulebookcompany.com/api/v1/fee-schedule-results?exchange_name=fee_nyse_arca_options&exchange_name=fee_cboe_us_bzx_options&fee_action=make&fee_participant=market_maker&latest_only=true&page_size=50" \
  -H "x-rulebook-api-access-key: YOUR_API_KEY"
```

**Penny ETF fees scraped in 2025, with source document:**

```bash
curl -X GET "https://api.rulebookcompany.com/api/v1/fee-schedule-results/?fee_symbol_classification=etf&fee_symbol_type=penny&start_date=2025-01-01&end_date=2025-12-31&include_source=true" \
  -H "x-rulebook-api-access-key: YOUR_API_KEY"
```

## Pagination

<Steps>
  <Step title="Make your first request">
    The response `meta` includes `record_count` and `total_pages` so you know the full dataset size.
  </Step>

  <Step title="Iterate through pages">
    Increment `page_number` from `0` to `total_pages - 1`:

    - Page 1: `?page_number=0&page_size=50`
    - Page 2: `?page_number=1&page_size=50`
    - Page 3: `?page_number=2&page_size=50`

  </Step>
</Steps>
